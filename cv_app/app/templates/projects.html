1. Gym Photo Classifier

This project implements a fully automated image classification pipeline using FastAI, designed as a backend component for a web application that identifies gym exercises and equipment from uploaded photos. Key features include:

  • Automated dataset preparation: the script prepares the dataset folder, renames image files to a standardized format, and generates the labels.csv file with appropriate multi-label annotations (e.g., ['squat', 'barbell']).
  • Multi-label classification support – each image is associated with exactly two labels.
  • Custom normalization pipeline – using manually calculated mean and std values for image preprocessing.
  • Efficient data handling – includes data augmentation, resizing, and usage of ResNet50 for transfer learning.
  • Evaluation with F1 Score – the model is assessed using the multi-label F1 score after test-time augmentation (TTA).
  • Model export (gym_model.pkl) – ready for integration into the web front-end.
This project was built specifically to support a custom web interface for recognizing gym-related content in user-uploaded images. It is fully extendable and can be further improved or tested as needed.

Technologies: Python Flask PyTorch Bootstrap

2.Sales Website with Artificial Neural Network for Demand and Profit Prediction

This project is a modern web application that employs an artificial neural network to analyze and predict sales data. The app retrieves data stored in an MSSQL database, displays the entire sales table through a Flask-based web interface, and then generates forecasts of future demand and potential profit based on the collected data. Predictions are visualized using dynamically generated Matplotlib charts, which are saved directly to an in-memory buffer — eliminating the need for static file storage and enhancing the app’s performance and flexibility. By leveraging a neural network model, the system learns from historical data and provides increasingly accurate sales forecasts, significantly aiding inventory optimization and strategic business planning. This project is an excellent example of integrating machine learning with a web application, enabling interactive data handling and real-time visualization of results directly within the browser.

3. Big Data Image Mosaic Generator Using Parallel Processing

This project is a hands-on demonstration of applying Big Data principles to image processing. The Python script generates a mosaic image by assembling thousands of small thumbnail images into a single, large, high-resolution picture. Each region of the input image is matched with the closest-fitting thumbnail based on color similarity. In our case, the dataset contained over 24,000 source images, which posed significant challenges in performance and memory usage. To address this, the script uses parallel data processing via Python's multiprocessing module and optimized matrix operations with NumPy. Project highlights:

  • Handles massive image datasets (24,000+ files)
  • Demonstrates practical use of Big Data in image analysis
  • Parallel processing significantly reduces execution time
  • Intelligent color matching to generate seamless mosaics
  • Custom techniques to manage high-volume visual data
This project illustrates how unconventional and performance-driven solutions can be applied to real-world problems involving high-throughput image processing and large-scale data handling.

Technologies: Python Pillow NumPy Multiprocessing

5. Web and Mobile Application with Flask & Firebase Real-Time Database

This project is a full-featured web application complemented by a dedicated mobile app designed to display and manage data seamlessly. Built with a Flask backend and integrated with Firebase’s Real-Time Database, it showcases an efficient use of a NoSQL data model tailored for dynamic, real-time updates. The application provides comprehensive CRUD (Create, Read, Update, Delete) operations on a custom-defined JSON-based data structure representing patient information. Unlike many typical implementations that overwrite entire data nodes during updates, this system smartly updates only specific fields within the patient’s JSON tree, preserving data integrity and preventing accidental data loss. By demonstrating how to work with a non-relational database in a practical healthcare context, the app highlights key benefits of Firebase Real-Time Database such as instant synchronization, flexible data modeling, and robust multi-platform support. The mobile app complements the web frontend by providing easy access to real-time patient data on the go.
